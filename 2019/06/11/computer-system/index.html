操作系统和计算机网络

一.知识点汇总


1.第一个常考点是进程和线程的区别和联系:
(1)进程是系统资源分配的最小单位,线程则是程序执行的最小单位.
(2)进程使用独立的数据空间,线程是共享进程的数据控件.

2.第二个常考点是线程的调度:
主要了解线程调度的算法:

  • 时间片轮转调度(RR)
  • 先来先服务调度(FCFS)
  • 优先线调度
  • 多级反馈队列调度(Multilevel feedback queue)
  • 高响应比优先调度

3.线程切换的步骤:
(1) 主要了解线程的上下文切换
(2)线程切换的代价

4.线程间通信(也就是IPC)
面试中间件研发的岗位会经常考察

需要了解:Pipe, Message Queue, 共享内存, Unix Socket, Signal和Semaphore.
这六种进程通信方式的原理和适用场景都需要有所了解.

  • 例如:进程间数据共享的场景可以使用共享内存.进程间进行数据交换的场景可以使用Unix Socket或者消息队列.

5.协程部分:
只需要了解协程更轻量化,是在用户态下进行调度,线程的切换比上下文切换的代价低很多.(也可以去了解Java的第三方框架如:Kilim和Quasar)

6.Linux命令主要考察是否有线上排查的经验.

7.内存分页管理和Swap,任务队列与CPU Load在分析线上问题时十分有用.
